<h1?GZ Stream Deck<h1/> 🎛️



GZ Stream Deck is a custom, fully open-source hardware and software ecosystem designed for streaming, productivity, and granular per-application audio control.

Built using an Arduino ATmega32U4 and Java 25, it bridges the gap between physical hardware and low-level Windows APIs, allowing you to mix individual application volumes (like Discord, Spotify, or your game) using physical sliders, while binding complex keyboard shortcuts to physical macro keys.

✨ Features

🎚️ Per-App Audio Mixing (WASAPI): Dynamically detects applications making sound and lets you route their specific volume to a physical hardware slider. No more tabbing out to adjust Discord!

⌨️ Macro Execution Engine: Bind any of the 5 physical buttons to complex keyboard combinations (e.g., Ctrl + Shift + M).

🧠 Smart Persistence: Automatically saves your channel mappings and macro bindings locally. Automatically re-maps applications when they are reopened.

⚡ Ultra-Low CPU Usage: Implements Delta-State Caching and background thread throttling. The app only recalculates data when you physically move a slider.

🎨 Sleek Custom UI: A beautiful, borderless dark-mode interface built with FlatLaf. Minimizes directly to the System Tray to keep your taskbar clean.

⚙️ True Native Application: Uses jpackage to compile into a standalone gzstreamdeck.exe with an embedded JVM. Displays correctly in Windows Task Manager.

🛠️ Hardware Requirements

Microcontroller: 1x Arduino Pro Micro or Leonardo (Must use the ATmega32U4 chip for native USB HID support).

Inputs: 5x Linear Slide Potentiometers (10kΩ recommended) & 5x Tactile Push Buttons.

Wiring:

Buttons: Connect to Digital Pins 2, 3, 4, 5, 6. The other leg goes to GND. (Internal pull-up resistors are used).

Sliders: Connect the middle wiper pins to Analog Pins A0, A1, A2, A3, A4. Outer pins connect to 5V and GND.

💻 Software Development Setup

Prerequisites

Arduino IDE: With the Joystick Library by M. Heironimus installed.

Apache NetBeans: Version 2025 or newer.

Java JDK 25: Installed and configured globally.

1. Flash the Firmware

Open the GZStreamDeck.ino file in the Arduino IDE and upload it to your ATmega32U4 board. This firmware includes built-in hysteresis noise filtering to prevent USB bus spamming.

2. Build the Java Application

The desktop application uses Maven to handle dependencies (hid4java, JNA, FlatLaf).

Open the project folder in Apache NetBeans.

Ensure your custom icon (app_icon.ico) is located in src/main/resources/.

Click the Clean and Build Project button (Hammer & Broom icon). Do not click the green "Run" button during the build phase.

3. Generate the Standalone Native EXE

To ensure the application shows up properly as gzstreamdeck in the Windows Task Manager (and not as "Java/Zulu Platform"), we compile it into a native C++ wrapper using jpackage.

After running "Clean and Build" in NetBeans, open the main project folder in Windows Explorer.

Double-click the provided build_native.bat script.

The script will bundle the Java Virtual Machine and your app into a standalone package.

Navigate to target\native-app\gzstreamdeck\ and launch gzstreamdeck.exe.

🚀 Usage Guide

Audio Mixer

Open the app and navigate to the Audio Mixer tab.

The dropdowns will automatically populate with active applications (an app must make a sound at least once to appear).

Select an application (e.g., Spotify.exe) for Channel 1.

Move the physical slider mapped to Channel 1. The volume indicator in the UI and the actual Windows volume for Spotify will adjust logarithmically for smooth fading.

Key Bindings (Macros)

Navigate to the Key Bindings tab.

Click inside one of the text boxes (e.g., Hardware Button 1).

Press your desired key combination on your keyboard (e.g., Ctrl + Alt + F9).

Pressing the physical hardware button will now instantly inject those keystrokes into Windows.

System Tray

Clicking the - (Minimize) button on the custom title bar will hide the application from your taskbar. A "GZ" icon will appear in your Windows System Tray (bottom right). Double-click the tray icon to restore the window.

🏗️ Architecture & Dependencies

hid4java: Used for asynchronous polling of the USB HID reports generated by the Arduino.

Java Native Access (JNA): Crucial for bypassing the JVM to interact directly with the Windows Component Object Model (COM). It utilizes custom VTable memory offsets to map IAudioMeterInformation and ISimpleAudioVolume to control per-app WASAPI audio streams.

FlatLaf: Provides the hardware-accelerated, borderless dark theme.
